What is JavaScript?

JavaScript is a programming language that enables you to create dynamically updating content, control multimedia, animate images, and much more. It is a core technology of the World Wide Web alongside HTML and CSS, and is supported by all modern web browsers.

Why learn JavaScript?

JavaScript is one of the most popular programming languages in the world. It is used to create interactive websites, web applications, games, and more. Learning JavaScript will open up a world of possibilities for you as a developer.

How to learn JavaScript?

There are many resources available to help you learn JavaScript, including online tutorials, books, and courses. You can also practice your skills by building projects and collaborating with other developers.

JavaScript Async Function

JavaScript Async

JavaScript is a single-threaded language, meaning it can only do one task at a time. However, to perform non-blocking operations like fetching data from a server or interacting with files, JavaScript supports asynchronous programming. This allows the code to execute without waiting for tasks like data fetching to complete, improving performance.

                
                    function fetchData(callback) {
                        setTimeout(() => {
                            const data = 'Data fetched!';
                            callback(data);
                        }, 2000);
                    }

                    function displayData(data) {
                        console.log(data);
                    }

                    fetchData(displayData);

Callback

A callback is a function passed into another function as an argument. It's called once the asynchronous task is completed. Callbacks were the first way of handling async code in JavaScript but often led to "callback hell" due to nested callbacks.

                
                    function greeting(name) {
                        console.log(`Hello, ${name}`);
                    }
                    
                    function processUserInput(callback) {
                        const name = "John";
                        setTimeout(() => callback(name), 1000); // async operation
                    }
                    processUserInput(greeting);

Promises

Promises provide a way to handle async operations more elegantly than callbacks.

                
                    let fetchData = new Promise((resolve, reject) => {
                    let success = true; // simulating success
                    if (success) resolve("Data fetched successfully!");
                    else reject("Error fetching data.");
                    });
                    
                    fetchData
                        .then((message) => console.log(message))
                        .catch((error) => console.log(error));

Async/Await

Async/Await is syntactic sugar built on Promises. It allows us to write asynchronous code in a more synchronous and readable manner. An `async` function always returns a Promise, and you can `await` a Promise inside it, meaning JavaScript will wait for that Promise to resolve or reject before moving on to the next line of code.

                
                    async function fetchData() {
                        try {
                            const message = await new Promise((resolve, reject) => {
                                setTimeout(() => {
                                    resolve("Data fetched");
                                }, 1000);
                            });
                            console.log(message);
                        } catch (error) {
                            console.error(error);
                        }
                    }
                    
                    fetchData();

Event Loop

The Event Loop is a mechanism in JavaScript that handles the execution of multiple pieces of code, allowing non-blocking, asynchronous operations to be processed. It constantly checks the Call Stack and the Message Queue. When the Call Stack is empty, the Event Loop picks the next function in the Message Queue to execute, ensuring async code runs when ready.

                
                    // Example to explain Event Loop
                    console.log('Start');

                    setTimeout(() => {
                        console.log('Timeout Callback');
                    }, 1000);

                    console.log('End');
